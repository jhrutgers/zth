3 # Zth (libzth) - Zeta threads
4 
5 This library provides user-space cooperative multitasking, also known as
6 fibers. One can see fibers as threads, but with the exception that you
7 explicitly have to indicate when the fiber is allowed to switch to another
8 fiber. As a result, locking, synchronization, using shared data structures
9 between fibers is way more easier than when using threads. See also
10 <https://en.wikipedia.org/wiki/Fiber_(computer_science)>.
11 
12 Check out the [examples](https://jhrutgers.github.io/zth/examples.html) to get
13 a grasp about how it works.
14 
15 A predecessor project was called Xi, as the Greek capital symbol suggests
16 parallel threads. In this project, preemptive multitasking is implemented. In
17 the same context, the Z(eta) symbol suggests that threads are not parallel, but
18 they explicitly yield from one to another.
19 
20 [GNU Pth](https://www.gnu.org/software/pth/) has been a great inspiration for this library.
21 
22 Currently, Linux/Mac OSX/Windows, 32/64-bit, gcc >=4 is supported.
23 There is experimental support for bare-metal 32-bit ARM (with newlib 3.1.0).

26 ## How to build
27 
28 When starting from scratch, clone the repository, like:
29 
30  git clone https://github.com/jhrutgers/zth.git
31  cd zth
32  git submodule init
33  git submodule update
34 
35 Build Zth out-of-tree:
36 
37  mkdir build
38  cd build
39  cmake ..
40  cmake --build .
41 
42 After building, check out the `doc` directory for documentation.
43 
44 For Mac OSX with MacPorts, one could change the sequence above by something like:
45 
46  CC=gcc-mp-8 CXX=g++-mp-8 LDFLAGS=-L/opt/local/lib cmake ..
47 
48 By default, release builds are generated. To do debug builds, do something like:
49 
50  cmake .. -D CMAKE_BUILD_TYPE=Debug
51 
52 For Windows with Qt, one could run cmake like this:
53 
54  set PATH=c:\Program Files\CMake\bin;%PATH%
55  cmake.exe .. -D CMAKE_BUILD_TYPE=Debug -G "Unix Makefiles" -D CMAKE_MAKE_PROGRAM="c:/qt/Tools/mingw730_64/bin/mingw32-make.exe" -D CMAKE_C_COMPILER="c:/qt/Tools/mingw730_64/bin/gcc.exe" -D CMAKE_CXX_COMPILER="c:/qt/Tools/mingw730_64/bin/g++.exe"

58 ## How to run
59 
60 libzth checks for environment variables, which are listed below. The
61 environment is (usually) only checked once, so dynamically changing the
62 variables after startup has no effect to libzth's behavior.
63 
64 * `ZTH_CONFIG_ENABLE_DEBUG_PRINT`
65  When set to 0, debug prints are suppressed. Enabled by default. For
66  non-debug builds, all debug prints are removed from the binary, so they
67  cannot be enabled in that case.
68 
69 * `ZTH_CONFIG_DO_PERF_EVENT`
70  When set to 1, perf VCD file generation is enabeled. Disabled by default.
71 
72 * `ZTH_CONFIG_DO_PERF_SYSCALL`
73  When set to 1, the perf VCD logs will contain all calls to Zth's special
74  functions. Enabled by default.
75 
76 * `ZTH_CONFIG_CHECK_TIMESLICE_OVERRUN`
77  When set to 1, check at every context switch if the timeslice was overrun
78  significantly. Only the longest overrun is reported. Enabled by default
79  in the debug build, not available in release builds.
